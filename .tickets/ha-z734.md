# Theme token cleanup: kill undefined CSS vars and hardcoded palettes

CONTEXT: theme.css defines the design tokens (--bg, --text, --primary, etc. + .light-theme and .high-contrast overrides), but key surfaces bypass them, so theming half-works by accident:
- src/routes/+layout.svelte lines 87-95 styles .app-shell with var(--bg-primary, hex) / var(--text-primary, hex) — variables that DO NOT EXIST anywhere; the shell is really painted by the hardcoded fallbacks, so high-contrast mode never changes the app frame.
- src/routes/+page.svelte uses nonexistent --text-secondary (lines ~306, 342, 417), --accent for CTAs while the Button component uses --primary (two different "primary" colors on one screen), and a hardcoded streak gradient.
- OfflineIndicator.svelte (lines ~52-103) and InstallPrompt.svelte (lines ~78-147) hardcode their own palettes (greens/reds/indigo gradient).
